If a repo exists, fetch && reset it. If it doesn't, clone it

This is currently very dumb in the sense that it only checks if a directory exists, not whether it's the git repo represented by self or even whether it's a git repository at all.

This is the equivalent of doing:

(test -d <local_path> && \
  cd <local_path> && \
  git fetch && \
  git reset --hard <ref>) || \
git clone --depth=1 --single-branch -b <ref> <remote_path> <local_path>
